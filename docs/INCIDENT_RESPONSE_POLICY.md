# PayGuard Incident Response Policy (IRP) - Draft

**Last Updated:** 2026-03-23

## 1. Detection
- Monitor service health, error spikes, auth failures, and abuse events.

## 2. Triage
- Classify severity (Sev1/Sev2/Sev3).
- Assign incident commander and communications owner.

## 3. Containment
- Revoke compromised keys/tokens.
- Apply temporary rate limits or endpoint controls.

## 4. Eradication and Recovery
- Patch root cause.
- Restore service and validate integrity.

## 5. Communication
- Notify affected customers based on severity and contractual obligations.
- Provide status updates and expected resolution timeline.

## 6. Postmortem
- Publish root cause, impact, timeline, corrective actions.
- Track action items to completion.
